MKT Helpdesk Agentic AI Automation
Technical Architecture & Operational Guide

Dokumen Resmi Presentasi Head Dept - Confidential

1. Pendahuluan & Gambaran Umum (Overview)

MKT Helpdesk Agentic AI Automation adalah platform resolusi IT tingkat lanjut (Next-Gen ITSM) yang menggabungkan sistem Ticketing tradisional, pemantauan perangkat keras proaktif (IoT/WMI Sensor), dan kecerdasan buatan (CrewAI). Sistem ini dirancang untuk mendiagnosa, menjawab, dan meresolusi kendala secara otomatis (Auto-Pilot) baik yang dilaporkan oleh Manusia (via Zammad & Telegram) maupun oleh Mesin (via Hardware Agent).

Arsitektur utama mengusung konsep Microservices yang di-containerization menggunakan Docker, memastikan skalabilitas, keamanan, dan kemudahan proses deployment di On-Premise maupun Cloud.

2. Arsitektur Sistem Secara Detail (Deep Dive)

Sistem ini terbagi menjadi 6 pilar utama yang saling berkomunikasi melalui protokol REST API dan Webhook:

2.1. Frontend (User Interface)

  • Teknologi: Vanilla HTML5, CSS3, JavaScript modern (ES6), dan Vite sebagai build tool.
  • Fitur Auto-Pilot (Real-Time): Menerapkan sistem Auto-Polling (melalui setInterval di app.js) yang menyegarkan data Dashboard dan Incident Memory setiap 10 detik tanpa *flicker* layar.
  • Sistem Notifikasi: Badge notification pintar yang mendeteksi data insiden baru menggunakan localStorage. Angka merah akan otomatis hilang saat halaman insiden dibuka.
  • Fungsi: Memvisualisasikan metrik KPI (Open Tickets, SLA, MTTR) dan menyajikan UI estetik berstandar tinggi untuk monitoring.

2.2. Backend API Server

  • Teknologi: Python 3.11 dipadukan dengan framework FastAPI yang asinkron (async/await).
  • Peran Utama: Menjadi jembatan utama (Gateway). Semua data masuk lewat sini sebelum diproses.
  • Zammad Polling Mechanism: Terdapat background task di zammad_webhook.py yang mem-poll tiket dari Zammad setiap 15 detik (diatur di ZAMMAD_POLL_INTERVAL pada config.py).
  • Hardware Webhook Receiver: Menyediakan endpoint /api/devices/{device_id}/hardware-alert untuk menerima status kerusakan (kabel LAN/USB tercabut) dari PC Kasir.

2.3. Telegram Bot Integration (Sistem Komunikasi)

  • Teknologi: Telegram Bot API (terintegrasi di backend dan CrewAI).
  • Peran: Menjadi alternatif pelaporan tiket bagi Kru/Kasir selain Zammad, sekaligus menjadi sistem *Push Notification* bagi tim MKT IT.
  • Alur Kerja (Workflow):
    1. Kru melaporkan masalah ke Bot Telegram Helpdesk.
    2. Pesan diteruskan ke Backend FastAPI melalui Webhook.
    3. AI Agent menganalisis masalah, dan AI dapat langsung membalas chat ke Kru via Telegram.
    4. Jika AI mendeteksi *hardware error* parah, AI akan menge-tag grup IT MKT di Telegram untuk eskalasi.

2.4. Agentic AI Engine (CrewAI + RAG)

  • Teknologi: CrewAI Framework, Gemini 2.0 Flash (LLM), ChromaDB (Vector Database).
  • Sistem Multi-Agen:
    • Classifier Agent: Bertugas membaca awal keluhan, menentukan tingkat urgensi (Severity), dan klasifikasi masalah (Hardware/Software/Network).
    • Knowledge Agent: Menggunakan tools KnowledgeSearch untuk membongkar ChromaDB mencari SOP atau rekam jejak penyelesaian masa lalu.
    • Resolution Agent: Mengeksekusi penyelesaian. Jika masalah tingkat dasar (Tier 0), ia memandu pelapor secara langsung. Jika masalah mendalam (Tier 1), ia mengeskalasikan masalah ke Human-in-The-Loop (HITL).
  • RAG (Retrieval-Augmented Generation): Menggunakan ChromaDB. Seluruh log historis Excel (1100+ baris) telah disaring dan di-*ingest* menjadi 21 Kasus Inti (Knowledge Base). AI tidak akan pernah "halu" karena jawabannya dibatasi hanya merujuk pada ChromaDB ini.

2.5. Database & Message Broker

  • PostgreSQL 16: Database relasional utama penyimpan Incident Memory, Data Zammad, Konfigurasi Telegram, dan *Audit Trails*. Sangat persisten karena menggunakan Docker Volumes.
  • Redis 7: In-Memory Store berkecepatan tinggi. Digunakan untuk sinkronisasi antrean tugas (Task Queueing) antara Backend dan AI, serta mem-bypass beban request berlebih.

2.6. Hardware Monitor Agent (Client-Side)

  • Teknologi: PowerShell Script (hardware_monitor.ps1).
  • Mekanisme: Skrip ini didaftarkan sebagai Windows Service (via install_service.bat) agar berjalan otomatis invisible saat PC Kasir/FNB menyala. Menggunakan WMI (Windows Management Instrumentation) untuk mendeteksi *Event* kabel USB / LAN terputus secara aktual, lalu menembak payload JSON ke Webhook Backend.

3. Panduan Operasional Sistem (Start-up & Usage)

A. Mengaktifkan Seluruh Infrastruktur

  1. Buka aplikasi Docker Desktop pada Windows Server, pastikan mesin berjalan.
  2. Arahkan Command Prompt/Terminal ke direktori D:\Helpdesk MKT Agentic AI Automation.
  3. Jalankan file eksekusi: start_all.bat.
  4. Skrip akan secara pintar melakukan *build* ulang *image* jika ada pembaruan kode, lalu mengaktifkan 5 kontainer utama: mkt_postgres, mkt_redis, mkt_chromadb, mkt_backend, dan mkt_crewai.

B. Menggunakan Dashboard (Frontend)

  • Akses URL: http://localhost:8000 atau alamat IP server (Contoh: http://192.168.1.10:8000).
  • Halaman tidak perlu di-refresh manual oleh staf MKT. Sistem Auto-Polling berjalan setiap 10 detik. Jika ada *hardware alert* dari site, lencana (badge) notifikasi angka berwarna merah akan langsung muncul di panel sisi kiri (Sidebar).

4. Manajemen Bug & Pemecahan Masalah (Troubleshooter Guide)

Buku manual wajib bagi PIC IT / Programmer untuk menganalisa masalah di masa depan:

Komponen / Gejala Akar Masalah (Root Cause) Cara Resolusi (Solusi Pemrograman)
Zammad: Tiket telat masuk / Webhook tidak merespon. Rate Limiting dari server Zammad, atau polling interval terlalu lama/terlalu cepat. Buka backend/config.py. Cek konstanta ZAMMAD_POLL_INTERVAL. Pastikan diset ke 15 detik. Restart backend dengan docker restart mkt_backend.
Frontend: UI berantakan atau data lambat (Tersendat). Browser menyimpan Cache dari CSS/JS versi lama, atau Endpoint Backend kelebihan beban. Lakukan Hard Refresh di browser (Ctrl+F5). Cek app.js, pastikan durasi setInterval tidak di bawah 10000ms.
Hardware Agent: Notifikasi LAN putus tidak muncul di Dashboard. Service PowerShell terhenti di PC kasir, atau IP Backend berubah. Buka PC Kasir yang bermasalah. Jalankan uninstall_service.bat lalu jalankan ulang install_service.bat sebagai Administrator. Pastikan URL Webhook di skrip PowerShell sudah sesuai dengan IP Server.
CrewAI / RAG: AI memberikan jawaban yang melenceng / Tidak tahu SOP terbaru. Vektor Database ChromaDB belum memiliki dokumen SOP versi terbaru yang baru saja disahkan. 1. Edit file IT_Helpdesk_Knowledge_Base.md dan tambahkan kasus baru.
2. Jalankan ulang skrip penanaman data: docker exec mkt_crewai python ingest_rag.py.
3. AI akan otomatis belajar dalam hitungan detik.
Telegram: Bot tidak membalas pesan. Token Bot Expired atau Webhook Telegram terputus oleh Firewall. Cek file konfigurasi .env di dalam folder Backend & CrewAI. Pastikan TELEGRAM_BOT_TOKEN valid. Cek log kontainer backend: docker logs mkt_backend.

5. Keunggulan Strategis (Elevator Pitch)

  1. Resolusi Detik Itu Juga (Zero Latency): Agen pintar (Script Powershell) mendeteksi perangkat fisik rusak secara instan tanpa perlu kasir mengetik keluhan. Begitu kabel lepas, sistem RAG langsung merekomendasikan solusi kabel *kendor*.
  2. Omnichannel Terpusat: Penyatuan Telegram, Zammad, dan Sensor Hardware ke dalam satu jalur urat nadi (FastAPI) yang dipantau oleh satu "Otak" (CrewAI).
  3. Pemisahan Otoritas (Front-Facing vs HITL): AI sangat cerdas mengetahui batasan; instruksi *remote server* akan diisolasi dan dieskalasikan diam-diam ke IT Pusat, sementara instruksi mudah (*Restart / Cek Kabel*) diberikan langsung ke Kasir.
Pernyataan Jaminan Arsitektur: Seluruh subsistem berjalan di atas arsitektur yang terisolasi sempurna. Kegagalan satu kontainer (misal: Zammad down) tidak akan membuat sistem Auto-Polling Hardware menjadi terhenti, berkat implementasi Asynchronous Gateway.
